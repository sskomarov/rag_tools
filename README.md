# RAG Tools
Набор инструментов и функций для использования при построения RAG-систем (паттерн Classic RAG).

## Описание
Представлен набор инструментов для использования при построения RAG-систем (паттерн Classic RAG) с помощью распространенных фреймворков (langchain, chonkie), оценки ретривера и генерации (метрики на основе n-грамм, семантические метрики).

## Документация по модулям

Ниже представлена документация по основным модулям RAG Tools.

### API
Описание доступных эндпоинтов, входных и выходных данных, форматы запросов/ответов.  
Подробнее: [API документация](./docs/ru/api.md)

### Метрики
Описание реализованных метрик качества LLM и ретривера:
- RAG: MRR, MAP, HIT@K, Recall@K
- LLM: BLEU, CHRF, TER 
  Подробнее: [Документация по метрикам](./docs/ru/metrics.md)

### Quickstart
Быстрый старт работы с RAG Tools: установка, требования и быстрая проверка работы.  
Подробнее: [Quickstart Guide](./docs/ru/quickstart.md)

## Описание коммитов
### Шаблон
Для оформления сообщения коммита следует использовать следующий шаблон:

    <тип> (<область действия>): <пояснения>

### Типы коммитов


| Тип         | Описание                                                        |
|-------------|-----------------------------------------------------------------|
| docs        | Обновление документации                                         |
| feat	      | Добавление нового функционала                                   |
| fix	      | Исправление ошибок                                              | 
| refactor    | Правки кода без исправления ошибок или добавления новых функций | 
| style	      | Правки по кодстайлу (табы, отступы, точки, запятые и т.д.)      |
| test	      | Добавление тестов                                               |
| ci          | Изменения в файлах конфигурации                                 |

### Пример коммита
